# XMacro Bridge 工作指引

## 项目目标与边界

本项目开发一款中文、离线、绿色便携的 Windows 宏转换工具。第一版聚焦键盘事件、鼠标事件、延时和嵌套宏展开，不转换 DPI、灯光、完整设备配置、XMBC 应用规则或层级系统。

雷云方向默认输出独立宏 XML；XMBC 方向默认输出可复制或保存的宏文本。不得直接修改用户原始宏文件。

## 开始任务前必须阅读

按以下顺序读取：

1. `docs/README.md`
2. `docs/01-产品需求规格.md`
3. `docs/02-系统架构设计.md`
4. `docs/04-转换规则.md`
5. `docs/06-安全与可靠性规范.md`
6. `docs/08-测试与验收标准.md`

具体格式工作还必须阅读 `docs/03-宏格式规范.md`；UI 工作必须阅读 `docs/05-UI设计规范.md`；发布工作必须阅读 `docs/09-构建发布规范.md`。

## 强制工作规则

- 解析器、统一模型、转换器、验证器和 UI 必须解耦，依赖方向遵循架构文档。
- 未知或无法转换的事件必须产生可见诊断，不得静默丢弃。
- 所有输入文件默认只读；输出使用临时文件写入，成功后再原子替换或另存。
- 新增或修改格式行为时，同步更新 `docs/03-宏格式规范.md` 和 `docs/04-转换规则.md`。
- 验证范围必须与变更风险成比例，默认采用“最小充分验证”，不得因少量 UI、文案、间距或样式调整重复运行完整测试、完整自包含发布和全量设计 QA。
- 小范围纯 UI/XAML 调整只运行受影响项目的基础编译，以及一项与改动直接相关的冒烟或实机检查；若编译和针对性检查通过即可结束。
- 只有核心模型、格式转换、安全与文件写入、跨模块接口、构建发布链路发生变化，或者用户明确要求正式交付时，才运行 `tools/test.ps1 -Configuration Release` 全量验证。
- 缺陷修复优先增加最小、直接的回归断言；纯视觉问题可使用针对性 WPF 冒烟或实机截图验证，不要求为每个像素级调整扩展完整测试套件。
- 完成任务前运行适用的最小编译、测试或静态检查，并在需要时记录结果；不得把“运行更多测试”本身视为完成质量的证明。
- 每次开发结束更新 `docs/项目状态.md`，并运行 `tools/devlog/Update-DevLog.ps1 -Mode Session` 记录完成事项与待办。
- 重大技术或兼容性选择必须在 `docs/decisions/` 新增 ADR，不得只保留在聊天记录中。
- 禁止提交密钥、用户隐私、机器专属配置、构建产物、临时文件或未经匿名化的完整敏感宏。
- Git 提交保持单一目的；后台脚本不得自动提交或推送。
- `我的宏/` 是用户数据目录，不得移动、重命名、修改或纳入自动测试。

## 完成定义

任务只有在代码、测试、相关文档、项目状态和开发日志全部同步后才算完成。若因环境限制无法验证，必须记录具体命令、失败原因、风险和后续解除方式。
