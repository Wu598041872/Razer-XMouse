# 宏格式规范

> 本文记录已由真实样本确认的结构。推测字段必须标注“待验证”，不得作为稳定导出契约。

## 1. 雷云独立宏 XML

根元素为 `<Macro>`，常见子元素为 `<Name>` 和 `<MacroEvents>`。事件结构：

输入接受 UTF-8（无 BOM 或 BOM）、UTF-16 LE BOM 和 UTF-16 BE BOM；XML 声明必须与实际字节编码一致。Windows-1252 等本地代码页和 UTF-32 明确拒绝。导出仍固定为 UTF-8 no-BOM。

- `Type=0`：延时，`Number` 使用秒制小数。
- `Type=1`：`Version=4` 原生 XML 的 `KeyEvent/Makecode` 直接保存 Windows 虚拟键码，例如 E=`69`、W=`87`；`State=0` 表示按下，`State=1` 表示释放。缺少 Version 的旧兼容样本仍按键盘扫描码读取，导出固定生成 Version 4 VK。
- `Type=2`：`Version=4` 原生 XML 使用 `MouseButton=0` 左键、`1` 右键。缺少 Version 的旧兼容输入按内容探测，仍兼容一基 `1` 左键、`2` 右键；`State=0` 表示按下、`State=1` 表示释放。导出固定生成 Version 4 零基 `0/1`。
- `Type=6`：字段结构复用。仅含 `Delay` 时表示 Synapse 4 毫秒延时，值为非负整数毫秒；含 `Number` 和 `LoopEvent/State` 时表示有限循环边界，`State=0` 为开始、`State=1` 为结束，`Number` 为正整数循环次数。`Delay` 与 `LoopEvent` 不得同时存在。
- `Type=7`：嵌套宏引用，已知字段包括 `MPIndex` 与 `guid`。独立 XML 根 `<Guid>` 是宏自身的真实引用标识，必须优先作为 `MacroDocument.Id`；仅在根 Guid 缺失或无效时才由文件内容生成稳定标识。`MPIndex` 为从 1 开始的宏位置，不是零基数组下标。
- `Type=actionBar`：编辑器元数据，不作为执行事件输出。

`Id`、`flag`、`selected`、`isPairing` 等字段需保留兼容形态，但不参与统一执行语义。

已知类型但字段缺失、数值无效、按钮编码未知或引用无效的单个事件转换为 `UnknownMacroEvent`，并产生 `RAZER_EVENT_INVALID`；同一宏中前后的有效事件继续保留。未知 `Type` 使用 `RAZER_EVENT_UNKNOWN`。原始事件仅保存在未知事件载荷中，不得作为诊断来源上下文。

延时秒值转换为整数毫秒；不能精确表示时按 AwayFromZero 舍入并产生 `RAZER_DELAY_PRECISION_LOSS` Warning。负延时不允许通过舍入变成零，按损坏事件隔离。

2026-07-12 后续实机导入提供了更强证据：桥接器输出的 `Makecode=18` 在 Synapse 4 中显示为 Alt，`MouseButton=1` 显示为右键；同机多份原生 `Version=4` XML 使用 `Makecode=69` 表示 E，并一致使用 `0/1` 表示左/右键。因此 Version 4 导入与导出固定采用 VK 和零基鼠标编码；早期无版本兼容文件仍单独保留旧解析路径。该结论由 ADR-0005 替代此前兼容导出决策。

2026-07-13 四份真实独立 XML 验证：父宏 `Type=7/guid` 分别精确匹配子宏根 `<Guid>`；旧实现忽略根 Guid 并把 `MPIndex=1` 当作零基索引，导致 GUID 缺失后错误回退到父宏自身并误报循环引用。修复后两条真实引用均能展开，根 Guid 缺失或无效的非引用宏仍保留内容哈希回退。

## 2. `.synapse4`

规范样本为 UTF-8 JSON，顶层已知字段包括 `productId`、`deviceName`、`category`、`profiles` 和 `macros`。只读导入器的外层容器同时接受 UTF-8 BOM、UTF-16 LE BOM 和 UTF-16 BE BOM；Base64 宏载荷保持严格 UTF-8（允许 UTF-8 BOM），不接受 UTF-16。`profiles` 与 `macros` 条目包含：

- `name`
- `payload`：Base64 编码的 UTF-8 JSON。
- `hash`：校验字段，算法尚未确认。

第一版只读取整包，不生成整包。宏载荷中包含 `guid`、`macroEvents` 和 `macroPlaybackOption`。内层事件同时支持 `Type=0/Number` 秒制延时、`Type=6/Delay` 整数毫秒延时，以及 `Type=6/Number/LoopEvent/State` 有限循环边界。

只读导入器对外层 JSON 和内层 Base64 JSON 分别设置大小与深度限制。单个损坏宏产生条目级诊断，不阻止同一整包内其他有效宏导入；单个已知事件字段损坏时转为 `UnknownMacroEvent` 和 `SYNAPSE4_EVENT_INVALID`，不丢弃包含它的宏。`hash` 当前只记录为未验证信息，不作为拒绝读取依据。

## 3. XMBC 配置文件

`XMBCSettings.xml` 与 `.xmbcp` 均为 XML 配置。宏通常位于各应用、层级和按钮元素中：

输入编码与雷云 XML 相同：接受 UTF-8（无 BOM 或 BOM）、UTF-16 LE BOM 和 UTF-16 BE BOM，并拒绝不匹配的 XML 声明、本地代码页及 UTF-32；输出完整配置不在第一版范围。

- `action=28` 表示模拟按键序列。
- `keys` 保存宏文本。
- `keyaction` 表示执行方式。
- `keyrepeat` 表示重复间隔。
- `active`、`blockmouse`、`randomisedelay` 为附加行为。

导入时只提取 `action=28`，名称按应用、层级、按钮和 `desc` 组合生成。第一版不导出完整 XMBC 配置。

配置导入器同时识别默认配置、`Application` 配置、附加 `Layer1`–`Layer9` 和 `chords` 内的宏。XMBC XML 中直接位于配置下的按钮属于第 1 层，`Layer1` 对应用户界面的第 2 层，以此类推。来源的 `keyaction`、`keyrepeat`、`active`、`blockmouse` 和 `randomisedelay` 作为元数据保留，不擅自换算播放模式。

## 4. XMBC 宏文本

用户提供的 XMBC 软件内说明已确认以下类别：

宏文本输入接受 UTF-8（无 BOM 或 BOM）、UTF-16 LE BOM 和 UTF-16 BE BOM；只对文件开头的 BOM 进行编码识别，正文中的 `U+FEFF` 不被静默删除。导出固定为 UTF-8 no-BOM。

- 辅助键：`CTRL/RCTRL`、`ALT/RALT`、`SHIFT/RSHIFT`、`LWIN/RWIN`、`APPS`；可组合，但只应用于下一个普通或扩展按键。
- 扩展键、方向键、`F1`–`F24`、音量、多媒体、浏览器和数字键盘标记。
- 鼠标左/右/中键、`MB4/XMB1`、`MB5/XMB2` 及对应 `D/U` 按下释放后缀。
- 滚轮与倾斜：`MWUP/MWDN/TILTL/TILTR`。
- 延时与按住：`WAIT/WAITMS`、随机 `WAITMS:x-y`、`HOLD/HOLDMS`。
- 原始键码：`VKC/EXT/SC/SCE`。
- 移动、剪贴板、程序、动作、Windows 消息、方案切换和触发条件标记。
- `PRESS/RELEASE` 只影响紧随其后的一个普通键、扩展键或扫描码；消费该目标后立即恢复普通点击模式。显式按下仍必须由后续显式释放配对。
- `HOLD/HOLDMS` 与一次性 `PRESS/RELEASE` 同时出现时不视为未知标记：在 `PRESS` 下延时位于目标按键按下之后，在 `RELEASE` 下延时位于目标按键释放之前。

当前实现支持普通字母/数字/空格、辅助键组合、扩展/方向/功能/音量/多媒体/浏览器/数字键盘、`VKC/EXT/SC/SCE`、已确认鼠标标记、固定/随机延时、`HOLD/HOLDMS` 和一次性 `PRESS/RELEASE`。移动、剪贴板、程序、动作、消息、方案与触发条件作为 XMBC 专属命令完整保留，可原样导回 XMBC；转换到雷云时产生阻断错误。

`MWUP/MWDN/TILTL/TILTR` 没有独立的 `D/U` 后缀，是单次原子动作。导入器将每个标记规范化为相邻、同按钮的 `Down/Up` 事件；导出器只允许该相邻配对折叠回一个标记。若两个事件之间存在延时、其他事件、错按钮释放或孤立释放，必须产生 `XMBC_EXPORT_ATOMIC_MOUSE_PAIR_REQUIRED`，不得重排或静默消费。

匿名化黄金样本 `samples/xmbc/extended-tags.txt` 组合覆盖辅助键、随机延时、`F24`、中键、`MB4`、滚轮、数字键盘、音量、多媒体、浏览器、扫描码、鼠标移动和方案切换。样本必须能完成 XMBC 文件导入、统一模型验证、XMBC 导出和再次导入的事件级语义往返。

2026-07-12 实机基线：XMBC 2.20.5 便携版“模拟按键”编辑器接受 `{PRESS}a{WAITMS:50}{RELEASE}a{WAITMS:10}{LMB}`，未显示语法错误，并在动作下拉框中保留该序列。验收后已将测试按钮恢复为“无修改”，未点击“应用”或“保存配置”。
