# 转换规则

## 1. 统一时间与事件

- 内部延时单位为整数毫秒，使用非负 64 位数值。
- 雷云秒制延时乘以 1000 后按 AwayFromZero 四舍五入转换；发生精度损失时产生 `RAZER_DELAY_PRECISION_LOSS` Warning。负延时不得通过舍入变成合法零延时。
- 雷云 `Type=6` 的 `Delay` 已是整数毫秒，直接进入统一模型，不再乘以 1000；缺失、负数、小数或溢出按损坏事件隔离。
- 雷云 `Type=6` 若含 `LoopEvent`，按 `State=0` 开始、`State=1` 结束解析有限循环；开始和结束的正整数 `Number` 必须一致。
- XMBC 没有等价循环标记，因此有限循环在导入阶段完整展开为线性事件，循环边界不进入统一模型。雷云 XML 导出同样输出展开后的线性结果。
- 循环支持嵌套，默认最大深度 32；展开后最多 100,000 个事件。未配对、次数不一致、非法次数、深度或事件数超限均产生阻断错误，不得静默忽略或部分导出。
- 空循环直接折叠为空序列，即使次数很大也不得按次数空转。
- 零延时保留，连续延时可以在用户明确执行“规范化”时合并，默认不合并。
- 按下和释放保持原始顺序，不自动重排。
- XMBC `{PRESS}`/`{RELEASE}` 是一次性前缀，只改变下一个键或扫描码的状态并立即清除。`HOLD/HOLDMS` 在普通点击下保持“按下 → 延时 → 松开”；与一次性 `PRESS` 组合时转换为“按下 → 延时”，与一次性 `RELEASE` 组合时转换为“延时 → 松开”，不得转成未知事件。

## 2. 键盘与鼠标

- 统一模型保存 Windows 虚拟键码及可选显示名称，不按当前键盘布局改写键码。
- 雷云独立 XML 导入先检查根元素 `Version`：Version 4 固定按 VK 键码和原生 `0=Left/1=Right` 鼠标编码解析；缺少 Version 的旧文件才按是否出现 `MouseButton=0` 在零基与兼容一基 `1=Left/2=Right` 间选择。不得逐事件混用两套含义。
- 点击是导入/编辑层便利表示，导出前展开为按下和释放。
- 未建立目标映射的键码或鼠标按钮产生 `Error`，不得静默跳过。
- 修饰键必须保留左右侧信息；目标格式无法表达时产生明确降级警告。

## 3. 嵌套宏展开

- 在全部输入完成后建立引用索引。
- 独立 XML 使用根 `<Guid>` 作为宏自身标识；根 Guid 缺失或无效时才使用文件内容生成稳定标识。
- 引用 GUID 精确匹配优先，`MPIndex` 只作兼容回退；`MPIndex=1` 表示可用宏列表第 1 项，解析时转换为内部索引 `0`。名称只用于用户提示，不作为默认唯一键。
- 用户可将时间线中的 `MacroReferenceEvent` 手动重新绑定到当前工作区的其他宏；绑定后写入目标宏 GUID、名称和当前工作区一基位置，并立即重新执行循环、缺失引用和状态验证。当前宏不得绑定自身；更深层循环仍由解析器阻断并显示引用链。
- 用户可在选中事件之后插入一个指向已导入宏的引用事件；导出时与自动识别的引用使用完全相同的递归展开规则，不生成另一套执行语义。
- 子宏在调用位置完整内联，重复调用重复内联。
- 展开结果移除编辑器元数据与引用事件，保留调用前后的延时。
- 缺失引用和循环引用为 `Error`，诊断必须包含引用链。
- 默认最大嵌套深度 32、展开后最大事件数 100,000；超限中止。

## 4. 雷云 XML 导出

- 输出单个 `<Macro>`，包含名称和线性 `MacroEvents`。
- 延时格式化为秒制，至少保留毫秒精度。
- 重新生成稳定递增的事件 `Id`，正确配对按下与释放。
- 不输出 `Type=7`；`actionBar` 仅按雷云导入所需的最小兼容结构生成。
- Version 4 输出将 Windows 虚拟键码直接写入 `KeyEvent/Makecode`，不转换为扫描码；E 写为 `69`、W 写为 `87`。
- 左键写为 `MouseButton=0`，右键写为 `MouseButton=1`，与 Synapse 4 原生 Version 4 XML 和本次实机导入显示一致。
- 根节点输出 `DelaySetting=0`、文档 GUID、`Version=4` 和 `MouseMoveType=none`，`actionBar/recordProfile` 输出 `mmtSetting=0`。
- 键盘和鼠标按下时分配事件 ID，释放时复用相同 ID；输出先在内存中完整生成并验证，失败时不写入目标流。
- 当前只导出已通过真实样本确认的雷云左键和右键编码，其他鼠标按钮产生阻断错误。
- 2026-07-12 用户原始 XML 已完成只读导入验证：`0/1` 左右键与次数 5 循环展开为 147 个执行事件，XMBC 导出无错误，输入哈希不变。

## 5. XMBC 文本导出

- 延时使用 `{WAITMS:N}`。
- 可安全表示的普通点击使用简写；长按或交叠按键使用明确的按下/释放序列。
- XML 特殊字符只在保存到 XML 场景转义；纯文本输出保持 XMBC 语法。
- 无法表达的事件阻止导出并显示具体位置。
- 当前导出器使用显式 `{PRESS}`/`{RELEASE}` 和鼠标 `D/U` 后缀保留按下与释放边界，并支持已确认的普通键、扩展键、功能键、多媒体键、浏览器键、数字键盘和鼠标标记。
- `{MWUP}`、`{MWDN}`、`{TILTL}` 和 `{TILTR}` 是原子动作：仅当统一模型中相同按钮的按下/释放事件相邻时输出一个标记。中间插入延时或其他事件会改变语义，必须以 `XMBC_EXPORT_ATOMIC_MOUSE_PAIR_REQUIRED` 阻断，禁止静默折叠。
- XMBC 辅助键导入时在目标键前按下、目标键后按逆序释放；`HOLD/HOLDMS` 在目标键按下与释放之间插入延时。
- 为无歧义导出独立的辅助键按下状态，使用 `{VKC:n}` 或 `{EXT:n}` 配合 `{PRESS}/{RELEASE}`，而不是误用“仅作用于下一个键”的辅助键标记。
- 随机延时和非键鼠命令在统一模型支持其语义前不得近似转换。
- `{PRESS}/{RELEASE}`、`WAITMS` 和左键简写组合已通过 XMBC 2.20.5“模拟按键”编辑器实际接受验证。

## 6. 状态验证

- 每个按键和鼠标按钮不得在未按下时释放。
- 同一输入在未释放时再次按下默认产生错误。
- 宏结束时仍处于按下状态产生错误，用户可在编辑器中补充释放事件。

## 7. 损坏事件隔离

- XML/JSON 整体不良构、超限或无法解码时拒绝整份输入并产生文件级诊断。
- 宏容器中的单个损坏宏条目只隔离该条目，同包后续宏继续读取。
- 已知事件类型字段损坏时保留其原始载荷为 `UnknownMacroEvent`，并继续解析同宏后续事件；事件顺序和序号不得因隔离而重排。
- 原始事件载荷不得写入 `ConversionDiagnostic.SourceContext` 或默认日志；诊断来源使用宏名或脱敏文件名。
- 导入达到事件上限时产生 `IMPORT_EVENT_LIMIT`，停止读取后续执行事件，并在上限内用 `UnknownMacroEvent(SourceType=import.event-limit)` 标记截断状态，确保验证器和导出器继续阻断不完整宏。

## 8. 时间线编辑

- 单项修改当前仅接受固定 `DelayMacroEvent`，输入必须是大于或等于 0 的整数毫秒；随机延时使用批量缩放，不把双边界静默合并为固定值。
- 当前插入入口在选中事件之后添加固定延时；未选择事件时追加到末尾。插入和复制不得使事件数超过 `MaximumEventsPerMacro`。
- 键盘插入接受十进制 Windows 虚拟键码 `0`–`255`、`Down/Up` 和独立的 `IsExtended`；不根据当前键盘布局改写键码，不根据显示字符推断左右修饰键。
- 鼠标插入覆盖统一模型中的左/右/中键、两个侧键、滚轮上下和倾斜左右，`Down/Up` 由用户分别选择。编辑器不自动生成配对释放；滚轮与倾斜导出 XMBC 时仍必须满足相邻同按钮配对规则。
- 键盘或鼠标按下插入后可以暂时形成未释放状态；即时验证必须显示诊断并禁用导出，插入对应释放或撤销后恢复。
- 复制保留事件完整语义并选择新副本；删除后优先选择原位置的下一事件，没有下一事件时选择前一事件。
- 上移和下移按用户可见时间线顺序交换相邻事件；结构编辑完成后统一将全部 `Sequence` 重新编号为从 0 开始的连续值，非结构编辑不主动重排。
- 复制、删除或移动可以产生按键状态错误；编辑结果必须立即验证并禁用危险导出，禁止为了保持可导出而静默调整其他事件。
- 多选复制保持选中事件的可见顺序并在最后一个选中事件后插入；多选删除原子移除全部选中事件；多选移动和拖拽先提取选中事件，再作为一个连续有序块插入目标位置。组操作统一重新编号，任一操作只产生一条撤销历史。
- 批量缩放同时处理固定延时和随机延时的最小/最大边界，百分比必须大于或等于 0；结果按 `AwayFromZero` 舍入到整数毫秒，事件顺序、序号和非延时事件保持不变。
- 缩放比例为 0 时保留零延时事件，不自动删除或合并事件。
- 任一原始延时为负数、随机延时最小值大于最大值、计算溢出或结果超出 64 位毫秒范围时整批拒绝，禁止只应用部分结果。
- 编辑通过替换工作区中的不可变宏快照完成，不修改导入对象或原文件；每次编辑、撤销和重做后立即重新验证导出安全状态。相邻相同事件交换等语义无变化操作不进入历史，并产生可见状态提示。

## 9. 时间线搜索

- 搜索仅作用于当前宏的内存展示行，匹配可见 `Sequence`、事件类型和详细信息，使用 `OrdinalIgnoreCase`，不受当前区域性影响。
- 搜索词最长 128 个字符；首尾空白不参与匹配，纯空白等同未搜索。
- 搜索不会修改宏、原文件、诊断或撤销/重做历史。上一项和下一项只更新当前选中事件，并在到达边界后循环定位。
- 结果计数使用“当前位置 / 总数”；尚未定位时当前位置为 0。切换宏、编辑、撤销或重做后必须基于新时间线重建结果。
