# UI 设计规范

## 1. 设计原则

界面采用苹果风格启发的简约视觉，但不复制 Apple 商标或专有素材。优先级依次为清晰、安全、可访问、克制动效。

## 2. 视觉基线

- 字体：`Segoe UI Variable`，回退 `Microsoft YaHei UI`。
- 基础间距：4、8、12、16、24、32、40。
- 卡片圆角 16–24，控件圆角 8–12。
- 浅色背景 `#F5F5F7`，主卡片白色；深色主题使用接近黑而非纯黑。
- 强调色默认使用系统蓝；错误、警告、成功不得仅靠颜色表达。
- 阴影轻量，避免大面积玻璃和影响性能的模糊。

## 3. 页面结构

- 当前工作区左侧集中放置文件/文件夹导入和宏列表；后续功能扩展后再引入导入、工作区、导出、设置导航。
- 主区域自上而下为当前宏与目标格式、事件时间线、兼容性与安全报告。
- 顶部显示当前宏、来源摘要、目标格式和离线处理状态。
- 导出按钮仅在没有阻断错误时可用，并显示禁用原因。

## 4. 交互

- 支持拖放文件、多个文件和文件夹，也提供标准文件选择器。
- 长任务显示进度并可取消，UI 线程不得执行文件解析。
- 删除、批量延时修改等可逆操作进入撤销栈；新编辑发生在撤销之后时必须清空重做分支。
- 覆盖已有输出前必须确认；默认建议生成不重名文件。
- 错误信息使用中文说明“发生了什么、影响什么、如何处理”。

## 5. 当前实现基线

- 主窗口最小尺寸为 `900 × 500`，默认尺寸为 `1280 × 820`；manifest 使用 PerMonitorV2。紧凑高度通过整页纵向滚动兜底，整页不得出现横向滚动；滚动内容高度绑定可视区域并设置最小值，使列表和时间线始终获得有限测量空间，不得破坏虚拟化。
- 已实现空状态、文件/文件夹选择、拖放、多宏选择、事件时间线、固定延时/键盘/鼠标参数化插入、任意事件复制/删除/上下移动、固定延时修改、固定/随机延时批量缩放、事件搜索与循环定位、20 步撤销/重做、诊断列表、目标格式选择、安全另存、进度和取消。
- 时间线编辑工具栏分为结构操作、键鼠参数和延时编辑三行，并在窄宽度下使用自动换行，不得引入整页横向滚动。新事件默认插入选中事件之后；选中固定延时后自动填入毫秒值。非法键码、非法延时、事件上限、边界移动、无变化操作、负比例和溢出通过状态 live region 给出中文原因，失败操作不得进入历史。
- 键盘区域依次提供十进制 VK、按下/释放、扩展键和插入；鼠标区域依次提供按钮、按下/释放和插入。控件不得把按下和释放自动合并为点击，也不得隐藏滚轮/倾斜所需的相邻配对要求。
- 搜索区提供最长 128 字符的关键词输入、上一个、下一个和“当前位置 / 总数”。输入采用短延迟实时匹配；按钮循环定位并将选中行滚入视图。计数使用固定 UIA 名称和 `Polite` live region，不得把搜索关键词拼入可访问名称。
- ViewModel 的只读状态绑定必须显式使用 `OneWay`；不得依赖控件默认绑定模式，以避免启动期运行时异常。
- 浅色、深色和高对比度主题使用同一组语义资源；应用只读跟随 Windows 应用主题，并在系统偏好变化后实时刷新。
- 诊断区域支持按严重级别和来源筛选，并按宏、文件或工作区上下文分组；错误优先显示，筛选结果必须同时显示“当前数量/总数量”。
- 深色主题避免纯黑大面积背景，主背景使用 `#1C1C1E`、卡片使用 `#2C2C2E`；高对比度主题优先使用 `SystemColors`，不得仅靠红黄绿表达状态。
- 浅色强调文字使用 `#005FC7`；深色强调文字使用 `#66B2FF`；主按钮填充与普通强调文字分离，确保白色按钮文字达到至少 4.5:1。
- 正文、辅助文字、强调文字、错误、警告、成功及主按钮文字必须使用 WCAG 相对亮度公式自动验证，对其实际背景的对比度不得低于 4.5:1；诊断文字必须与 `DiagnosticSurfaceBrush` 配对验证。
- 主按钮的 hover/pressed 状态使用独立不透明填充色，不得通过降低整个按钮透明度改变文字对比度。键盘焦点使用与普通按钮或主按钮背景至少 3:1 的独立焦点环。
- 宏列表、时间线、目标格式、诊断筛选、诊断分组、进度和主要操作必须具有明确的 `AutomationProperties.Name`。
- 时间线和诊断区域各保留一个 Tab 入口，时间线内部使用方向键选择整行；编辑框和编辑按钮按“时间线 → 插入/复制/删除/移动 → 键盘参数 → 鼠标参数 → 单项延时 → 批量缩放 → 撤销/重做”的顺序进入 Tab。时间线单元格和诊断项不得逐项形成 Tab 陷阱。
- 宏、事件与诊断生成项的可访问名称不得退化为 record `ToString()`，不得包含 GUID、内部类型名、绝对路径或事件原始载荷。宏名和诊断来源经统一脱敏后用于名称；事件名称只包含序号和受控类型，诊断项只包含代码和级别。导入、导出和取消状态使用 `Polite` live region。

## 6. 可访问性

- 键盘可遍历所有操作，焦点样式清晰。
- 正文对比度至少 4.5:1。
- 支持 Windows 文本缩放和 125%–200% DPI。
- 尊重系统减少动画、高对比度和浅/深色偏好。
