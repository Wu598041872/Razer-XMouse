# UI 设计规范

## 1. 设计原则

界面采用接近 Razer Synapse / 雷云的暗色电竞设备控制台视觉，定位为紧凑、高信息密度、克制的 Windows 专业工具。优先级依次为清晰、安全、可访问、性能和品牌一致性，不使用渐变、玻璃拟态、大面积阴影或过度发光。

## 2. 视觉基线

- 标题优先使用 `RazerF5, Roboto, Helvetica Neue, Arial, Noto Sans SC`，正文与控件使用 `Roboto, Helvetica Neue, Arial, Noto Sans SC`，路径、序号和时间使用 `Cascadia Mono, Consolas`；缺失字体必须自然回退。
- 间距采用 8、12、16、20、24、30 DIP；正文 13–14 DIP，辅助文字 12 DIP，标题 16–18 DIP。
- 控件和面板圆角统一为 3 DIP；普通按钮、输入框和下拉框高度 34–36 DIP，时间线行最小高度 44 DIP，宏库条目最小高度 64 DIP。
- 注册色仅使用背景 `#161616`、表面 `#222222`、主文字 `#FFFFFF`、次要文字 `#999999`、边框 `#333333`、强调绿 `#44D62C` 和悬停绿 `#44F62C`；仅时间线鼠标事件允许使用 `#FF8A00`。
- 绿色只用于主操作描边与文字、选中、成功、焦点和关键状态；不使用大面积绿色填充、装饰渐变、红色危险按钮或额外类别色。
- 全局 `TextBlock` 必须保留 `TextPrimaryBrush` 作为普通暗色背景文字色；绿色高亮黑字只能通过 Button、ComboBoxItem、MenuItem 等局部控件模板绑定自身 `Foreground` 实现，不得移除或削弱全局普通文字色。
- 错误、警告和信息必须配合图标与明确文字，不得仅靠颜色表达。
- 应用只提供固定 Razer 品牌暗色风格，不随 Windows 浅色、深色或高对比度设置切换。

## 3. 页面结构

- 顶部使用 44 DIP 的 Razer Synapse 风格一体化自绘标题栏，包含 22×22 正式产品 Logo、`XMACRO BRIDGE`、宏转换/宏库/设置固定槽位页签及最小化/最大化/关闭按钮。
- 自绘标题栏必须通过 `WindowChrome` 保留原生窗口拖动、双击最大化、从最大化拖动还原、边缘缩放、PerMonitorV2 和系统任务栏边界；交互按钮需显式允许在标题区域命中，且不得进入工作区 Tab 顺序。
- 标题栏导航页签必须保持相同高度、外边距、内边距和边框厚度；切换页面只允许改变背景、边框、文字和图标颜色，不得让页签重新测量或改变底边基线。
- 76 DIP 二级工具栏展示产品说明、当前宏与未保存状态、目标格式、保存到宏库和安全导出。
- 二级工具栏右侧使用统一两行栅格：字段标签位于第一行，当前宏值与目标格式使用 36 DIP 字段，保存和导出使用 40 DIP 操作，并共享底边。
- 宏转换页采用 214–238 DIP 左栏加弹性编辑区；右侧依次为标题/搜索、上下文工具栏、按需展开的添加操作区、事件时间线、诊断报告和 34 DIP 状态栏。
- 设置页使用“宏库 / 转换与导出 / 工作区 / 关于”四类导航，所有开关和选择都绑定真实持久化设置；关于页低调标注“个人本地工具，非 Razer 官方产品”。
- 默认窗口为约 `1265 × 813 DIP`，最小尺寸 `900 × 500 DIP`；紧凑高度通过工作区纵向滚动兜底，不出现整页横向滚动。

## 4. 控件与弹层

- Button、TextBox、ComboBox、CheckBox、ListBox、DataGrid、ContextMenu、MenuItem、ToolTip、ScrollBar、ProgressBar 和 GridSplitter 必须使用统一资源样式。
- 所有可交互控件覆盖 normal、hover、pressed、focused、selected 和 disabled；焦点使用细绿色边框，不显示默认蓝色焦点框。
- 下拉弹层、右键菜单、工具提示、宏文本、重命名、事件编辑和应用消息对话框全部使用暗色主题。文件/文件夹/保存窗口保留 Windows 原生实现。
- 右键菜单与“更多”弹层必须完全接管 `ContextMenu` 和 `MenuItem` 模板，不得显示 WPF 默认白色图标槽、滚动槽或系统阴影；固定少量操作的菜单不得出现无用滚动区域。
- 滚动条宽度约 14 DIP，轨道接近面板背景，滑块固定为清晰可操作的 48 DIP 长度，默认 `#666666`，悬停变亮，拖动时使用绿色；紧凑窗口不得按内容比例将滑块压缩或裁切成圆点。
- 主要按钮使用暗色背景、绿色 1 DIP 描边和绿色文字；次要按钮使用暗色背景、白色文字和弱边框；焦点使用不改变布局的外置 2 DIP 绿色焦点环。

## 5. 时间线与编辑

- 时间线继续使用虚拟化 `DataGrid`，但隐藏表头并使用单个全宽模板列；主视觉只显示类别图标、按下/松开箭头和事件名称，技术详情放入工具提示。
- 键盘、鼠标、延时、未知/错误事件分别使用绿色键盘、橙色鼠标、灰色时钟和红色错误图标；键盘只显示键名，鼠标显示“左键单击”等动作，固定延时统一显示三位秒数，例如 `0.330s`。
- 行悬停只提升表面与边框明度；选中行显示完整绿色边框，整行文字和事件图标转为绿色。
- 每行固定显示复选框、两位可见序号、类别图标、方向箭头和事件内容；仅选中行显示复制、删除和六点拖动手柄。
- 复选框、序号、类别图标和方向箭头使用固定列宽并分别水平居中；不同事件类型不得因矢量边界差异产生左右跳动。
- 按下/释放方向不得使用细字体字符直接渲染；使用 Tabler `arrow-bar-to-down` 与 `arrow-bar-up` 轮廓图标，16 DIP、约 2.2 DIP 线宽，并保持同一中心线。
- 多选复制、删除、上下移动和拖拽作为有序块执行；拖拽只能由专用手柄发起。所有组操作进入同一撤销历史并立即标记未保存、重新验证导出安全。
- 拖拽排序必须提供明确落位反馈：目标行上半区显示绿色顶部插入线，下半区显示绿色底部插入线；鼠标附近显示绿色浮层“移动 N 个操作”。无效落点不显示插入线，离开时间线、取消或完成拖拽后必须立即移除全部反馈。
- 上下文工具栏只保留编辑、复制、删除、上下移动、添加操作、撤销和重做；延时、键盘和鼠标参数仅在“添加操作”展开后显示。
- 键盘参数输入使用按键捕获：输入框获得焦点后直接按下目标键即写入 Windows VK；右侧修饰键、导航键和其他已确认扩展键自动勾选扩展状态。
- 键鼠参数行提供“嵌套宏”目标下拉框，列出当前工作区除当前宏之外的已导入宏；选中时间线中的嵌套宏事件后，“绑定宏”将其目标改为下拉选项，“插入宏”在当前位置之后新增引用。标签、下拉框与按钮作为整体换行，紧凑宽度下不得拆散。
- 嵌套目标下拉框宽度为 240 DIP；选择后仅在该控件上将所选宏的 `Name` 单向绑定到收起区域的 `Text`，长名称使用全局模板既有省略号并在悬停时显示完整名称。不得为此修改全局 ComboBox 对 `DisplayName` 选项的呈现行为。
- 固定延时、键盘和鼠标事件支持工具栏、右键菜单和双击打开统一暗色编辑对话框。编辑保持原事件序号，使用不可变宏快照进入撤销历史，并立即重建验证与导出安全状态。
- 未知、随机延时及暂不支持的事件不得静默近似修改；编辑入口应禁用，原事件数据保持不变。
- 5000 行时间线的已实现行数必须低于 200；工作区滚动测量不得破坏行虚拟化。
- 时间线从最大化或默认宽度缩至最小窗口时，唯一事件列必须同步收缩并保持横向偏移为 0；事件序号、图标、方向和名称不得因动态调整窗口而消失。
- 紧凑窗口下工作区内容尺寸必须扣除自身外边距，右侧搜索状态、筛选器、滚动条和取消按钮均需完整位于可视视口内。固定少量操作的右键菜单不得显示滚动槽或浅色系统阴影。

## 6. 报告、状态与可访问性

- 兼容性与安全报告使用深色分组面板，保留诊断数量、严重级别/来源筛选、空状态、操作状态、绿色进度和取消。
- 宏列表右键保留“重命名…”、“修改宏文本…”和“删除”；右键落点必须先成为当前选中项。
- 长任务不得阻塞 UI 线程，必须支持进度和取消；错误信息需说明发生了什么、影响什么以及如何处理。
- 正文与实际背景对比度至少 4.5:1，焦点边界至少 3:1；自动门禁覆盖固定 Razer 暗色风格。
- 支持 PerMonitorV2、125%–200% DPI 和中文文本缩放；不得出现乱码、文字裁切、组件重叠或明显溢出。
- 时间线和诊断区域各保留一个主要 Tab 入口，单元格和诊断项不得形成 Tab 陷阱；其他编辑控件按视觉顺序排列。
- 宏、事件与诊断生成项的 UIA 名称不得包含 GUID、内部类型名、绝对路径或原始事件载荷。导入、导出、取消和搜索状态使用 `Polite` live region。

## 7. 宏库

- 顶部导航顺序固定为“宏转换、宏库、设置”，三个页面使用各自二级工具栏，不显示与当前页面无关的转换操作；三枚页签切换前后 X 坐标、宽、高和底边应保持稳定。
- 宏库使用三列布局：左侧分组、中间统一宏列表、右侧单一活动宏详情与只读内容预览；任何时刻只能有一个活动宏。
- 分组区包含全部、收藏、最近使用、未分组、自定义分组和回收站；列表项显示名称、分组、修改时间或大小，并以绿色星标表示收藏。
- 选中条目后延迟解析并显示内部宏名、事件数和诊断状态；损坏文件保持可见，禁止静默隐藏。
- 统一宏列表显示 `XM` / `R4` 格式标记、名称、分组、修改时间和收藏状态；搜索、分组、格式筛选与排序可组合使用并尽量保持原选择。
- 统一宏列表上方固定提供“全部 / XMouse / 雷云 4”三个格式分类；分类计数、行内事件数和详情事件数必须来自真实解析结果，不使用文件大小或演示数据代替。
- 三个格式分类等宽占满中栏分类行，分类行禁用水平与垂直滚动条，不得显示无内容对应的多余滑块。
- 详情内容预览必须支持 XMouse 文本和雷云 4 XML 原文；预览读取不得复用仅允许 `.txt` 的编辑接口，解析失败时仍保留可读取的原始内容并显示明确状态。
- XMouse 新建/编辑对话框的多行文本区域必须占用主要可用高度、自动换行且不出现横向滚动条。
- “更多”操作保持扁平暗色菜单；当前菜单模板不允许使用无法完整渲染的嵌套子菜单。
- 新建、导入、打开转换、编辑/复制文本、打开位置、收藏和更多操作必须直接可见；移动、重命名和删除可放入紧凑菜单。

## 8. 渐进式工作区与空状态

- 无选中宏时隐藏搜索、插入、编辑、报告和分隔条，时间线区域显示“选择文件”和“输入宏文本”双入口；不得以整排禁用按钮代替操作引导。
- 已选择宏后显示延时、键盘、鼠标插入和撤销/重做；未选事件时显示“选择事件后可编辑”，选中后显示“已选 N 项”及编辑、复制、删除、上下移动。
- 只有工作区存在可引用的其他宏时才显示嵌套宏工具，避免单宏工作区占用额外高度。
- 报告区区分“未发现问题”和“当前筛选没有结果”；后者必须提供重置筛选操作。
- 操作状态使用独立 38 DIP 状态栏；进度条和取消按钮仅在忙碌时显示，不挤占报告内容。
- 时间线与宏库搜索框统一使用搜索图标和提示文字；`Enter` / `Shift+Enter` 定位下一个 / 上一个事件结果，`Esc` 清空搜索。

## 9. 操作层级与弹窗

- 次要按钮悬停只提升暗色表面与边框亮度，绿色只用于主操作、焦点、选中和成功反馈。
- 删除、永久删除和清空操作正常状态保持低强调；悬停使用绿色强调，但文案必须明确动作后果，并在执行前显示暗色确认框。
- 所有应用弹窗使用暗色 DWM 标题栏、应用图标、20 DIP 外边距和 32 DIP 控件高度；原生文件选择、文件夹选择与保存窗口除外。
- 表单错误必须同时提供图标与文字，并通过 UIA live region 通知；显示错误后焦点移至首个需要修正的字段。
- 基础快捷键固定为 `Ctrl+O` 选择宏文件、`Ctrl+F` 聚焦当前页面搜索、`Ctrl+Z` / `Ctrl+Y` 撤销与重做；文本框聚焦时保留原生文本撤销与重做。
