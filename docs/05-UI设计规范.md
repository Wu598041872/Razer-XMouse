# UI 设计规范

## 1. 设计原则

界面采用苹果风格启发的简约视觉，但不复制 Apple 商标或专有素材。优先级依次为清晰、安全、可访问、克制动效。

## 2. 视觉基线

- 字体：`Segoe UI Variable`，回退 `Microsoft YaHei UI`。
- 基础间距：4、8、12、16、24、32、40。
- 卡片圆角 16–24，控件圆角 8–12。
- 浅色背景 `#F5F5F7`，主卡片白色；深色主题使用接近黑而非纯黑。
- 强调色默认使用系统蓝；错误、警告、成功不得仅靠颜色表达。
- 阴影轻量，避免大面积玻璃和影响性能的模糊。

## 3. 页面结构

- 当前工作区左侧集中放置文件/文件夹导入和宏列表；后续功能扩展后再引入导入、工作区、导出、设置导航。
- 主区域自上而下为当前宏与目标格式、事件时间线、兼容性与安全报告。
- 顶部显示当前宏、来源摘要、目标格式和离线处理状态。
- 导出按钮仅在没有阻断错误时可用，并显示禁用原因。

## 4. 交互

- 支持拖放文件、多个文件和文件夹，也提供标准文件选择器。
- 长任务显示进度并可取消，UI 线程不得执行文件解析。
- 删除、批量延时修改等可逆操作进入撤销栈。
- 覆盖已有输出前必须确认；默认建议生成不重名文件。
- 错误信息使用中文说明“发生了什么、影响什么、如何处理”。

## 5. 当前实现基线

- 主窗口最小尺寸为 `1040 × 680`，默认尺寸为 `1280 × 820`；布局使用星号尺寸和可滚动列表，不依赖固定屏幕分辨率。
- 已实现空状态、文件/文件夹选择、拖放、多宏选择、只读事件时间线、诊断列表、目标格式选择、安全另存、进度和取消。
- ViewModel 的只读状态绑定必须显式使用 `OneWay`；不得依赖控件默认绑定模式，以避免启动期运行时异常。
- 浅色、深色和高对比度主题使用同一组语义资源；应用只读跟随 Windows 应用主题，并在系统偏好变化后实时刷新。
- 诊断区域支持按严重级别和来源筛选，并按宏、文件或工作区上下文分组；错误优先显示，筛选结果必须同时显示“当前数量/总数量”。
- 深色主题避免纯黑大面积背景，主背景使用 `#1C1C1E`、卡片使用 `#2C2C2E`；高对比度主题优先使用 `SystemColors`，不得仅靠红黄绿表达状态。
- 浅色强调文字使用 `#005FC7`；深色强调文字使用 `#66B2FF`；主按钮填充与普通强调文字分离，确保白色按钮文字达到至少 4.5:1。
- 正文、辅助文字、强调文字、错误、警告、成功及主按钮文字必须使用 WCAG 相对亮度公式自动验证，对其实际背景的对比度不得低于 4.5:1。
- 宏列表、时间线、目标格式、诊断筛选、诊断分组、进度和主要操作必须具有明确的 `AutomationProperties.Name`。

## 6. 可访问性

- 键盘可遍历所有操作，焦点样式清晰。
- 正文对比度至少 4.5:1。
- 支持 Windows 文本缩放和 125%–200% DPI。
- 尊重系统减少动画、高对比度和浅/深色偏好。
