# 安全与可靠性规范

## 1. 离线与隐私

- 核心功能不得发起网络请求。
- 日志默认只记录格式、事件数量、耗时、诊断代码和脱敏文件名，不记录宏全文。
- 不采集遥测、账号、设备序列号或应用使用数据。

## 2. 输入保护

- 输入文件以只读共享方式打开，不写回、不改名、不移动。
- 不信任扩展名；先探测内容和结构，再选择解析器。
- 禁止 XML 外部实体、DTD 和不受控实体展开。
- JSON、Base64、XML 深度、文件大小、事件数量和嵌套深度均设置上限。
- 文本探测只接受 UTF-8 和带 BOM 的 UTF-16 LE/BE；UTF-16 无 BOM 不猜测，UTF-32 明确拒绝。完整解码使用严格错误回退，探测前缀允许被 4096-byte 边界截断但不得改变原始输入流。
- 默认单文件上限 64 MiB、批次 512 MiB、单宏 100,000 事件、嵌套深度 32；超限可在设置中降低，不允许无限制。
- 直接输入的 XMBC 文本不得落地为临时输入文件；在内存中按 UTF-8 计算字节数并应用同一 512 MiB 批次上限，再进入既有 XMBC 导入器与事件上限。
- Razer XML、`.synapse4`、XMBC 文本和 XMBC 配置必须在各自解析循环内执行事件上限；单个标记短暂展开后立即裁剪，返回模型不得超过上限。

## 3. 输出安全

- 输出先写入同目录随机临时文件，刷新并关闭后再原子提交。
- 默认不覆盖；覆盖必须由用户确认并保留失败恢复路径。
- 输出文件名清除 Windows 非法字符、保留名和尾随点空格。
- `CON/PRN/AUX/NUL/CLOCK$`、`COM1`–`COM9`、`LPT1`–`LPT9` 等 Windows 保留名称必须在创建临时文件前拒绝。
- 目标被外部进程独占时，原目标内容必须保持不变，失败结果使用 `EXPORT_FILE_ERROR`，临时文件与备份文件必须清理；锁释放后同一服务实例应能成功重试。
- 导出器在写入部分临时内容后抛出 `IOException` 时不得提交目标；父路径组件是普通文件而非目录时不得创建额外产物或改变该文件。
- 应用退出或取消时清理自身临时文件，不扫描或删除其他文件。

## 4. 稳定性

- 解析与导出支持取消和超时。
- 所有资源通过确定性释放关闭，异常不得遗留输入文件锁。
- 导入取消门禁必须在输入流已经打开后触发，取消完成后原文件 SHA-256 保持不变、能够以 `FileShare.None` 重新打开，并允许同一服务实例再次导入。
- 导出取消门禁必须在同目录临时文件已经写入部分内容后触发；既有目标内容不得改变，临时文件和备份文件必须清理，目标文件锁必须释放，工作区恢复可导出状态并允许立即重试。
- UI 全局异常只负责记录和友好退出，不吞掉可恢复业务错误。
- 编辑操作使用不可变快照或命令模式，确保撤销/重做一致。
- 当前工作区撤销历史最多保留 20 步；重新导入会清空历史。批量延时缩放必须先完整计算并验证全部结果，再一次性替换宏快照，禁止溢出时留下部分修改。
- 插入和复制必须在创建快照前检查事件数量上限；结构编辑必须一次性规范化顺序并替换完整快照。复制、删除和移动造成的按键状态错误由即时验证阻断导出，不得自动补键或静默丢弃事件。
- 键盘 VK 输入只接受 `0`–`255`；界面按键捕获通过 WPF `KeyInterop` 生成 VK，无法映射的键不修改输入。非法值不创建事件或历史。键鼠插入不自动补充配对事件，用户明确插入的 `Down/Up` 原样进入统一模型，并由状态验证决定导出安全性。
- 最大 100,000 事件时间线的单次搜索自动门禁要求 5 秒内完成，当前线程瞬时分配低于 1,000,000 bytes；序号匹配不得为每个事件创建临时字符串。
- 长时间编辑回归在 5,000 事件下连续执行 100 轮搜索、删除、撤销、重做和再次撤销，要求 30 秒内完成，完整 GC 后保留内存增长低于 64 MiB；搜索结果、选择和动态诊断不得跨快照失效或累积。

## 5. 供应链

- 依赖版本固定并定期审计；能用 BCL 完成的格式解析不引入额外包。
- 发布构建生成 SHA-256，保留依赖清单和构建环境版本。
- 不运行输入文件中包含的命令、路径、脚本或宏事件。

## 6. 系统集成边界

- 应用不读取或修改 Windows 主题偏好，不订阅系统主题或高对比度变化。
- UI 筛选只改变内存中的展示结果，不修改诊断、宏文档或输入文件。
- 时间线搜索只读取当前展示行的序号、受控类型名和详细摘要；关键词不得写入日志、诊断、UIA 名称或撤销历史。搜索索引必须在宏切换和时间线编辑后重建，禁止持有失效事件引用。
- `XMACROBRIDGE_TEST_MODE=1` 仅用于隔离自动化宏库目录，不改变产品界面样式。
- UI Automation 名称不得包含输入文件绝对路径、宏原始内容、GUID 或内部对象完整表示。用户提供的宏名与来源先移除绝对路径和 GUID，并限制长度；事件和诊断名称只使用受控摘要字段。
- `UnknownMacroEvent.RawPayload` 只用于显式兼容性检查，不得复用为诊断来源上下文、日志字段或 UIA 名称。
- 文本输入框内容不得进入日志、状态文本或 UI Automation 名称；关闭对话框后不在 App 层保留额外副本。
- 为支持右键修改，XMBC 原始文本只保存在当前内存 `MacroDocument.Metadata` 的受控字段中；不得写入日志、诊断、UIA 名称或临时文件，重新导入或关闭进程后自然释放。
- 文件级 `ConversionDiagnostic.SourceContext` 只保存 `Path.GetFileName` 形式的脱敏文件名，不保存调用方传入的绝对路径。

## 7. 宏库存储安全

- 应用不得自动发现或扫描仓库、安装目录或名为“我的宏”的目录；只有用户设置的宏库根路径可以进入扫描范围。
- 普通宏操作只接受根目录或一级分组内的 `.txt/.xml` 相对路径，拒绝绝对路径、路径穿越、重解析点和 Windows 保留名。
- 外部导入默认复制且不改动源文件；文本保存、设置和元数据使用同目录临时文件后提交。
- 删除先移动到 `.xmacrobridge/trash` 并记录原相对路径；迁移只接受空目标目录，逐文件 SHA-256 校验全部成功后才切换设置，失败时当前根路径和源目录保持不变。
- 收藏与最近使用只记录相对路径和时间，不记录宏正文；自动化测试使用系统临时目录，禁止读取用户真实宏库。
