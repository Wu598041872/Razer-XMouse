# 测试与验收标准

## 0. 验证范围原则

- 日常开发默认采用“最小充分验证”，不以全量测试次数衡量质量。
- 小范围纯视觉调整（文案、间距、颜色、对齐、边框、圆角、图标尺寸）只要求 App 基础编译加一项直接相关的 WPF 冒烟或实机目视检查。
- 局部 UI 交互或绑定变化只运行直接相关的测试；不得无理由触发格式、核心压力、发布产物和完整设计 QA。
- 核心模型、格式转换、安全、文件写入、跨模块契约和发布链路改动按受影响范围运行测试，必要时逐级扩大。
- 完整 `tools/test.ps1 -Configuration Release` 只在高风险跨模块改动、阶段性里程碑、正式测试包或正式发布前运行，或由用户明确要求时运行。
- 小改动不默认生成自包含测试包、ZIP、哈希和完整截图对照；仅在用户需要可执行版本或交付节点时生成。
- 同一轮后续修改只允许维护一个可覆盖更新的测试构建；所有针对性检查都在该构建上继续进行，不连续生成带不同时间戳的测试目录、ZIP 或 SHA-256 文件。
- 测试构建更新后必须删除或覆盖上一份，验收目录中只保留当前测试版本；正式发布包和用户明确要求保留的对比版本除外。
- 若最小验证失败、发现关联回归或影响范围无法确定，必须升级验证范围。

## 1. 测试层级

- 单元测试：事件模型、键码、延时、状态验证、引用图算法。
- 格式测试：每个导入器和导出器使用最小黄金样本。
- 集成测试：导入 → 展开 → 验证 → 导出的完整路径。
- UI 测试：导入、选择、编辑、诊断、取消和导出关键流程。
- 编辑测试：固定延时修改必须保持原导入对象不变；固定/随机延时缩放验证 `AwayFromZero` 和溢出整批拒绝；结构编辑验证插入位置、复制/删除选择、上下移动、连续序号、事件上限、语义无变化操作、撤销/重做前后选择、分叉编辑清空重做、20 步历史上限、重复宏 GUID 隔离和即时重新验证；参数化插入覆盖 VK 边界、扩展键、全部鼠标按钮、按下/释放平衡和 XMBC 原子滚轮配对；搜索覆盖序号/类型/详情、大小写不敏感、128 字符上限、无结果、双向循环、编辑后重建和不进入撤销历史。
- 重命名测试：验证右键菜单与对话框可访问性、空名称拒绝、首尾空白修剪、128 字符上限、工作区不可变替换、撤销/重做，以及雷云 XML `<Name>` 使用重命名后的值且不保留旧名称。
- 事件显示测试：至少断言 E/W、左 Ctrl、方向键、鼠标左/右键和侧键的事件类型同时包含具体按键或按钮及“按下/松开”；技术详情继续包含 VK 或按钮枚举，搜索“键盘”等类别词保持兼容。
- 键盘捕获测试：在真实 WPF 窗口向按键输入框发送 `PreviewKeyDown`，验证 E 捕获为 69 且不启用扩展键，右 Ctrl 捕获为 163 并启用扩展键；事件必须标记为 Handled。
- 工具栏简化测试：全部延时缩放与选中延时应用控件必须为 `Collapsed`，其他结构编辑、键鼠插入和搜索控件保持可用和对齐。
- 暗色工作区测试：时间线编辑、键鼠插入、缩放、搜索和撤销/重做工具必须可见；时间线和诊断区由可访问的行方向 `GridSplitter` 相邻调整，并保持各自最小高度和紧凑窗口纵向滚动兜底。
- 宏文本修改测试：验证右键菜单、原文预填、重新解析、宏名与 ID 保留、撤销恢复、诊断重建和非 XMBC 宏的安全文本生成。
- `PRESS/RELEASE` 一次性作用域测试：覆盖 `{PRESS}e`、`{RELEASE}e` 后紧跟多个普通点击，后续键必须生成完整 Down/Up，禁止产生 `KEY_UP_WITHOUT_DOWN`。同时覆盖 `PRESS` 后 HOLD 延时和 `RELEASE` 前 HOLD 延时，禁止产生 `XMBC_TOKEN_UNKNOWN`。
- 启动冒烟测试：Release 程序必须实际创建主窗口，不得仅以编译成功代替 XAML 运行时验证。
- 发布产物测试：`tools/test.ps1` 必须生成真实的 Windows x64 自包含 App 到系统临时目录，直接读取 `XMacroBridge.App.exe` 的默认 `RT_MANIFEST` 资源，并确认 `dpiAwareness` 唯一且等于 `PerMonitorV2,PerMonitor`；验证结束后删除临时发布目录。
- 诊断界面测试：验证严重级别筛选、来源筛选、分组、空状态及“当前数量/总数量”。
- 界面测试：应用始终加载唯一的 Razer 品牌暗色色板，不再测试浅色或高对比度主题分支。
- 自动化 WPF 冒烟通过命令行导入匿名化样本，验证主窗口、宏列表、事件时间线、文件与文本导入入口、导出/取消状态、诊断筛选、诊断分组和主题资源；不得读取“我的宏”目录。
- 完整测试仅启动一次固定 Razer 暗色真实 WPF 窗口；使用 WCAG 相对亮度公式检查关键语义色与实际页面、卡片、次级、诊断、选中和按钮背景，并验证主要控件具有可访问名称、按钮和下拉框可进入键盘焦点。
- 主按钮 normal/hover/pressed 文字对比度不得低于 4.5:1；普通按钮和主按钮焦点环分别对实际背景达到至少 3:1。禁止通过整体透明度实现按压状态。
- WPF 冒烟使用固定的四个匿名化文件清单，不依赖共享 `samples/` 目录总数量；新增格式样本不得无关地破坏 UI 门禁。
- 宏库回归使用独立临时目录，覆盖未分组/一级分组、附件与深层目录忽略、复制哈希、UTF-8 原子文本保存、重名编号、收藏/最近、重命名/移动、回收/恢复、损坏设置和迁移校验；不得读取真实“我的宏”目录。
- WPF 冒烟必须切换“宏转换、宏库、设置”三个页面，验证对应内容和工具栏互斥、宏库搜索/刷新可用，以及测试宏库根路径位于系统临时目录。
- 正式窗口运行时必须使用 PerMonitorV2 DPI context。`900 × 500 DIP` 合同要求存在纵向滚动兜底且无整页横向滚动。
- 自动检查只读时间线单元格不进入 Tab、诊断区域只有一个入口、所有生成项名称无内部类型、GUID 或绝对路径、状态文本为 `Polite` live region；测试注入带匿名化绝对路径和 GUID 的恶意摘要验证脱敏。
- 5,000 事件时间线冒烟要求已实现的 `DataGridRow` 少于 200，防止外层滚动破坏虚拟化。100,000 事件搜索回归要求结果准确、可定位最后一项、单次搜索少于 5 秒且当前线程分配低于 1,000,000 bytes。
- 长时间状态回归使用 5,000 事件执行 100 轮“搜索最后一项 → 删除 → 撤销 → 重做 → 撤销”，每一步比较事件数、选择和结果计数；总耗时少于 30 秒，完整 GC 后保留内存增长低于 64 MiB，诊断集合不得增长。
- 取消测试使用受控阻塞导入器和导出器，不依赖文件大小或机器速度：分别在输入流已打开、临时输出已写入后发出取消，验证文件句柄、输入哈希、目标原内容、临时文件清理、工作区状态和同实例重试。
- 导出异常测试覆盖目标 `FileShare.None` 独占锁、导出器部分写入后抛出 `IOException`、父路径组件为普通文件；均验证 `EXPORT_FILE_ERROR`、原对象保护、无临时产物，并在可恢复场景解除条件后重试。
- 实机验收：雷云 4 导入 XML、XMBC 接受生成的宏文本。

## 2. 必测场景

- 字母、数字、功能键、左右修饰键和组合键。
- 鼠标左右键及已确认支持的扩展按钮。
- 零延时、长延时、连续延时和精度舍入。
- 多层嵌套、重复引用、缺失引用、循环引用和深度超限。
- 未按下释放、重复按下、宏结束未释放。
- 损坏 XML/JSON/Base64、错误编码、空文件、超大文件和超长宏。
- 直接输入 XMBC 文本必须与同内容 `.txt` 产生相同统一事件；空白输入不可提交，超出批次字节上限产生 `IMPORT_BATCH_SIZE_LIMIT`，且不得创建临时文件。
- 雷云已知事件的缺字段、坏数值、未知鼠标按钮、坏状态和无效引用必须只隔离当前事件；同包坏宏不得阻止前后有效宏。
- 四类外层输入分别覆盖 UTF-8 BOM、UTF-16 LE BOM 和 UTF-16 BE BOM；UTF-32 必须可见拒绝，`.synapse4` 的 Base64 内层保持严格 UTF-8。
- XML 编码声明必须与 BOM/UTF-8 实际编码一致，Windows-1252 等本地代码页不得被底层 XML 解析器绕过门禁。
- 文件级格式诊断的来源上下文不得包含绝对路径；导入前后输入 SHA-256 必须一致。
- 精确整数毫秒不产生告警，半毫秒及其他非整数毫秒产生 `RAZER_DELAY_PRECISION_LOSS`。
- 雷云 XML 与 `.synapse4` 的 `Type=6/Delay=265` 必须解析为 265 ms，工作区允许导出；XMBC 输出包含 `{WAITMS:265}`，雷云 XML 再导入后延时保持不变。
- 雷云 XML 与 `.synapse4` 的 `Type=6/Number/LoopEvent` 有限循环必须按次数完整展开；单层和嵌套循环比较事件数量、顺序、连续序号及 XMBC/雷云线性导出结果。
- 雷云独立 XML 必须覆盖 Version 4 的 VK `Makecode=69/87` 和原生 `MouseButton=0/1` 左右键，并覆盖无 Version 旧兼容文件的扫描码与 `1/2` 读取；Version 4 导出必须断言 E/W、左右键原生代码及再次导入语义。
- 循环缺少开始或结束、开始/结束次数不一致、次数非正整数、嵌套深度超限及展开事件数超限必须产生阻断错误，并保留可见未知事件；超大次数空循环必须在固定资源内折叠为空。
- 取消解析/导出、目标重名、磁盘写入失败和只读目录。
- 取消导入后原输入必须可独占打开且哈希不变；取消覆盖导出后目标必须保持原内容，目录中不得残留 `.tmp` 或 `.bak`，随后重试必须能原子替换目标。
- 目标被占用或写入中断时，既有目标内容必须逐字节保持；父路径不是目录时阻断文件也不得变化。所有失败路径结束后目录内只能保留调用前已存在的项目。
- 使用小型注入上限验证四类导入路径均在解析中停止；使用默认 100,000 上限运行大输入压力回归，返回模型不得超限且必须保留阻断哨兵。
- 固定延时从非法值修复后必须立即清除动态阻断诊断并恢复导出；撤销后诊断和阻断状态必须同步恢复，重做后再次清除。
- 复制或移动按键事件产生状态失衡时必须立即禁用导出；撤销后恢复原顺序、选择和诊断。达到事件上限时插入与复制必须拒绝且不创建历史。
- 键盘按下插入后产生 `KEY_NOT_RELEASED`，插入匹配释放后清除；鼠标执行相同平衡检查。`-1`、`256` 和非数字 VK 必须拒绝且不修改快照。插入相邻滚轮 `Down/Up` 后 XMBC 导出必须恢复原子标记。
- 时间线搜索在尚未定位、有结果、无结果和清空时必须显示正确计数；上一项/下一项按钮循环选择匹配行并滚入视图。搜索词不得出现在结果计数的 UIA 名称中，编辑、撤销和重做后不得保留失效结果。
- 最大 100,000 事件时间线必须完整建立展示行并搜索最后一个序号；压力回归先预热 JIT，再测量单次搜索耗时和当前线程分配，避免把启动成本误判为搜索成本。
- 重复编辑压力结束后必须回到原始不可变宏对象、最后一项选择和单一搜索结果；删除及重做阶段不得保留已删除事件的搜索索引。

## 3. 质量门禁

- Release 编译无警告；警告视为错误。
- 正式 App 自包含发布成功，且嵌入 Manifest 自动检查通过；只检查源 `app.manifest` 或运行时窗口状态不能替代发布产物检查。
- 变更代码路径有对应自动化测试。
- 缺陷修复先有失败回归测试。
- 未知事件产生结构化诊断。
- 格式导入器已经为未知或损坏事件提供根因错误时，工作区不得为同一来源、同一序号追加重复的通用 `UNKNOWN_EVENT`。
- 输入哈希在测试前后相同，证明原文件未被修改。
- 关键转换比较事件数量、顺序、状态平衡和总时长。

## 4. 发布验收

- 在干净的 Windows 10 和 Windows 11 x64 环境解压运行。
- 不安装 .NET SDK、不联网、不使用管理员权限也能启动和转换。
- 自包含 `XMacroBridge.App.exe` 内嵌 `RT_MANIFEST` 保持 `PerMonitorV2,PerMonitor`。
- 雷云 4 与 XMBC 实机验收样本全部通过。
- 无高优先级崩溃、数据损坏、静默丢事件或隐私泄漏问题。
