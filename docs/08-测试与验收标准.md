# 测试与验收标准

## 1. 测试层级

- 单元测试：事件模型、键码、延时、状态验证、引用图算法。
- 格式测试：每个导入器和导出器使用最小黄金样本。
- 集成测试：导入 → 展开 → 验证 → 导出的完整路径。
- UI 测试：导入、选择、编辑、诊断、取消和导出关键流程。
- 启动冒烟测试：Release 程序必须实际创建主窗口，不得仅以编译成功代替 XAML 运行时验证。
- 诊断界面测试：验证严重级别筛选、来源筛选、分组、空状态及“当前数量/总数量”。
- 主题测试：至少验证当前 Windows 主题下的实际启动；发布前补充浅色、深色和高对比度三种模式的视觉与可访问性验收。
- 自动化 WPF 冒烟通过命令行导入匿名化样本，验证主窗口、宏列表、事件时间线、导出/导入/取消状态、诊断筛选、诊断分组和主题资源；不得读取“我的宏”目录。
- 完整测试分别以浅色、深色和高对比度色板启动真实 WPF 窗口，使用 WCAG 相对亮度公式检查关键语义色至少达到 4.5:1，并验证主要控件具有可访问名称、按钮和下拉框可进入键盘焦点。
- 实机验收：雷云 4 导入 XML、XMBC 接受生成的宏文本。

## 2. 必测场景

- 字母、数字、功能键、左右修饰键和组合键。
- 鼠标左右键及已确认支持的扩展按钮。
- 零延时、长延时、连续延时和精度舍入。
- 多层嵌套、重复引用、缺失引用、循环引用和深度超限。
- 未按下释放、重复按下、宏结束未释放。
- 损坏 XML/JSON/Base64、错误编码、空文件、超大文件和超长宏。
- 取消解析/导出、目标重名、磁盘写入失败和只读目录。

## 3. 质量门禁

- Release 编译无警告；警告视为错误。
- 变更代码路径有对应自动化测试。
- 缺陷修复先有失败回归测试。
- 未知事件产生结构化诊断。
- 输入哈希在测试前后相同，证明原文件未被修改。
- 关键转换比较事件数量、顺序、状态平衡和总时长。

## 4. 发布验收

- 在干净的 Windows 10 和 Windows 11 x64 环境解压运行。
- 不安装 .NET SDK、不联网、不使用管理员权限也能启动和转换。
- 雷云 4 与 XMBC 实机验收样本全部通过。
- 无高优先级崩溃、数据损坏、静默丢事件或隐私泄漏问题。

## 5. 当前实机基线

- 2026-07-12：本机 Razer Synapse 4 成功导入 `basic-key-delay.xml`，宏列表出现“匿名化基础宏”。
- 2026-07-12：XMBC 2.20.5 便携版接受基础 `{PRESS}/{RELEASE}`、`WAITMS` 和 `{LMB}` 组合文本；未保存测试配置。
- 上述基线仅覆盖匿名化基础键鼠序列，扩展键、随机延时、滚轮、扫描码和目标软件版本升级仍需独立验收。
- 自动化扩展黄金样本覆盖随机延时、扩展键鼠、滚轮、扫描码及 XMBC 专属命令，并比较导出再导入后的完整事件签名。
