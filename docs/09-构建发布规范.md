# 构建发布规范

## 1. 版本

使用语义化版本 `MAJOR.MINOR.PATCH`。预览版本追加 `-alpha.N`、`-beta.N` 或 `-rc.N`。版本号、变更日志和程序集信息必须一致。

## 2. 环境

- .NET 8 SDK，版本由后续 `global.json` 锁定。
- Windows x64 构建主机。
- App 与 WPF 冒烟必须使用同一份 PerMonitorV2 `app.manifest`。
- Release 构建启用确定性编译、Nullable 和警告即错误。

## 3. 标准命令

```powershell
./tools/test.ps1 -Configuration Release
./tools/release/Build-Portable.ps1
```

当前两个自动化项目是离线控制台测试宿主，必须由 `tools/test.ps1` 通过 `dotnet run` 执行；单独运行 `dotnet test` 不构成质量门禁。脚本负责 restore、Release 构建、核心断言、Windows x64 自包含临时发布、嵌入 Manifest 检查、单次固定 Razer 风格 WPF 冒烟和日报回归。

发布产物必须通过 `tools/release/Assert-AppManifest.ps1` 检查嵌入 Manifest 仍声明 `PerMonitorV2,PerMonitor`，不得被发布参数或打包工具覆盖。校验器使用 `LoadLibraryEx` 的只读资源模式打开 EXE，只读取默认 `RT_MANIFEST`，不执行发布程序代码。`tools/test.ps1` 在系统临时目录生成并验证真实自包含产物，最终清理该目录。

正式引入 NuGet 依赖后提交锁文件；CI 和发布使用 `--locked-mode`。

## 4. 绿色包

ZIP 根目录包含程序、`README.md`、许可证、版本说明和 `Data/` 说明。用户数据、开发日志、测试样本和源码不进入发布包。

生成 ZIP 后创建同名 `.sha256` 文件。发布前在干净目录执行首次启动、导入、编辑、导出和完整删除测试。

`Build-Portable.ps1` 从 App 项目读取唯一版本号，默认先执行完整 Release 门禁，再生成 Windows x64 自包含 ZIP。脚本只在仓库 `artifacts/` 内创建和清理临时目录，不自动提交或打 Git 标签。

`Assert-PortablePackage.ps1` 必须验证 SHA-256、EXE `ProductVersion`、嵌入 Manifest、必要说明文件及 `Data/` 说明，并拒绝 PDB、源码、测试、样本、开发日志或用户宏目录进入发布包。

## 5. 回滚

- 每个正式版本保留不可变构建产物、SHA-256 和对应 Git 标签。
- 出现数据安全问题立即停止分发并回滚到最近通过验收的版本。
- 不在旧发布包上就地替换文件；修复必须发布新版本。
