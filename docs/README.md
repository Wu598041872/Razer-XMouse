# 项目文档索引

本目录是 XMacro Bridge 的工程事实来源。代码行为与文档冲突时，应先确认正确行为，再在同一变更中同步二者。

| 文件 | 用途 |
|---|---|
| `01-产品需求规格.md` | 产品目标、范围、流程和验收条件 |
| `02-系统架构设计.md` | 模块边界、依赖方向、数据流和错误模型 |
| `03-宏格式规范.md` | 雷云与 XMBC 的已知文件及事件格式 |
| `04-转换规则.md` | 统一事件模型、嵌套展开和导出规则 |
| `05-UI设计规范.md` | 视觉、布局、交互和可访问性标准 |
| `06-安全与可靠性规范.md` | 输入保护、资源限制、日志脱敏和恢复策略 |
| `07-开发工作流程.md` | 从领取任务到提交和日志记录的流程 |
| `08-测试与验收标准.md` | 测试层级、门禁和发布验收 |
| `09-构建发布规范.md` | 版本、构建、绿色包和回滚 |
| `10-开发路线图.md` | 分阶段交付物和退出条件 |
| `11-用户使用说明.md` | 绿色版启动、导入、编辑、导出和安全使用说明 |
| `项目状态.md` | 当前待办、进行中、完成、阻塞和风险 |
| `decisions/` | 架构决策记录（ADR） |

## 维护规则

- 文档使用 UTF-8 和简体中文，代码符号及格式字段保留英文。
- 重大决策新增 ADR；既有 ADR 不改写结论，只新增替代 ADR。
- 格式行为变化必须同时更新格式规范、转换规则和测试。
- 文档不得包含用户宏全文、个人路径、账号、密钥或设备唯一标识。
