# ADR-0001：使用 .NET 8 与 WPF

- 状态：已接受
- 日期：2026-07-11

## 背景

产品仅面向 Windows 10/11 x64，需要离线文件处理、成熟桌面控件、绿色自包含发布和较低的系统集成风险。

## 决策

使用 .NET 8、WPF 和 MVVM。转换核心位于不依赖 WPF 的类库中，UI 只负责展示和交互。发布使用 `win-x64` 自包含绿色包。

## 理由

- WPF 在 Windows 桌面环境稳定，文件拖放、DPI、键盘访问和主题定制成熟。
- .NET 提供安全 XML/JSON、Base64、异步、取消和文件 I/O 能力。
- Core 与 UI 解耦后可进行快速单元和黄金样本测试。
- 相比 WinUI 3，WPF 的绿色部署和运行时行为更可控。

## 影响

- 应用不支持非 Windows 系统。
- 需要自行实现苹果风格设计系统，不能依赖系统默认控件外观。
- 构建机必须安装并锁定 .NET 8 SDK；用户机器无需安装 SDK。
