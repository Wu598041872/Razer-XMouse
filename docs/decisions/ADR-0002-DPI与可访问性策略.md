# ADR-0002：使用 PerMonitorV2 与可持续可访问性合同

- 状态：已接受
- 日期：2026-07-12

## 背景

应用需要在 Windows 10/11 的 125%–200% 缩放、多显示器和高对比度环境中工作。仅依赖默认 WPF 行为会得到 System DPI awareness，固定最小窗口也会在高缩放的小有效工作区中被裁剪。只检查控件 `Focusable` 无法发现只读表格单元格形成的 Tab 陷阱或 UI Automation 泄漏绝对路径。

## 决策

- 应用 manifest 声明 `PerMonitorV2,PerMonitor`，运行时门禁要求 `DPI_AWARENESS=2`。
- 主窗口最小尺寸为 `900 × 500 DIP`，内容在紧凑高度下使用整页纵向滚动兜底，禁止整页横向滚动。
- 外层滚动内容高度绑定可视区域并保留最小高度，使时间线和宏列表始终得到有限测量空间；5,000 事件门禁要求已实现行数保持在 200 以下。
- 复合控件采用单一 Tab 入口，内部使用方向键或滚动操作；按钮和筛选器使用显式 `TabIndex`。
- 宏、事件、诊断和状态使用简短 UI Automation 名称，用户来源字段统一脱敏，禁止暴露内部类型、GUID、绝对路径或事件原始载荷；状态变化使用 `Polite` live region。
- 浅色、深色和高对比度分别运行真实 WPF 冒烟，验证实际前景/背景、按钮交互状态、焦点环及全部生成项容器。

## 影响

- WPF 冒烟项目与正式 App 共用 manifest，并需要在 Windows 桌面会话中运行。
- 紧凑窗口可能出现纵向滚动，但操作不会被裁剪或要求横向滚动。
- 自动合同不能替代跨显示器拖动、真实 Narrator 播报和多种物理 DPI 的最终人工验收。
