# ADR-0003：将雷云有限循环线性展开

- 状态：已接受
- 日期：2026-07-12

## 背景

Synapse 4 使用 `Type=6` 表示两种不同结构：`Delay` 字段表示毫秒延时，`Number` 与 `LoopEvent/State` 表示有限循环边界。XMBC 模拟按键语法没有等价的有限循环标记，静默丢弃边界会让转换结果只执行一次并改变宏行为。

## 决策

导入雷云 XML 或 `.synapse4` 时，根据字段结构区分延时与循环。`LoopEvent/State=0` 为开始，`State=1` 为结束，开始和结束的正整数 `Number` 必须一致。有效循环在格式适配器内按次数完整展开，统一模型和两类输出只接收线性事件。

支持嵌套循环，并使用 `MacroLimits.MaximumNestingDepth` 与 `MaximumEventsPerMacro` 限制资源。未配对、次数不一致、非法次数或资源超限产生阻断错误和可见未知事件。空循环直接折叠为空，不按次数执行空迭代。

## 理由

- 线性展开能在 XMBC 缺少循环语法时保持有限循环的执行次数和事件顺序。
- 循环边界不进入公共模型，避免 UI、编辑器、验证器和导出器分别实现一套循环状态机。
- 在导入边界执行资源门禁，可以在生成大量事件前安全停止。

## 影响

- 大次数循环会增加导入后的事件数量，并可能因安全上限被拒绝。
- 雷云 XML 再导出不会保留原循环结构，而会得到行为等价的线性宏。
- 无限循环或播放模式级重复不在本决策范围内，遇到未知结构仍必须阻断。
