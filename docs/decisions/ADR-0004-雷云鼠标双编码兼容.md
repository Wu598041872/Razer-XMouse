# ADR-0004：区分雷云鼠标输入与导入编码

- 状态：已接受
- 日期：2026-07-12

## 背景

Synapse 4 对独立宏 XML 使用了两套鼠标按钮编号。用户提供的原始 XML 与事件编辑器截图确认，原生文件中 `MouseButton=0/1` 分别表示左/右键；此前的导入实测又确认，提交给 Synapse 4 的兼容 XML 中 `0` 会显示为未选择操作，`1/2` 才分别显示左/右键。

## 决策

雷云 XML 导入器按整份文档识别编码：发现任一 `MouseButton=0` 时，文档按原生零基 `0=Left/1=Right` 解析；否则按兼容一基 `1=Left/2=Right` 解析。编码选择在解析事件前完成，同一文档不得逐事件切换含义。

雷云 XML 导出器继续固定输出 Synapse 4 导入界面已确认的 `1=Left/2=Right`。`.synapse4` 内层格式保持既有独立映射，不由本决策联动修改。

## 理由

- 同时兼容 Synapse 4 原始导出文件和 XMacro Bridge 生成的可导入文件。
- 文档级判断避免同一个代码 `1` 在同一宏中被解释成不同按钮。
- 保持导出编码稳定，避免再次生成在 Synapse 4 中显示“选择操作”的按钮事件。

## 影响

- 含代码 `0` 的真实原生文件可无损识别左右键并继续展开循环。
- 仅包含代码 `1` 且没有 `0/2` 的外部文件无法单凭字段判断是原生右键还是兼容左键；当前为保证本工具输出往返而按兼容左键处理。
- 中键、侧键及其他代码仍需真实匿名化样本确认，未知代码继续阻断导出。
