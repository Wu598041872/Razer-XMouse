# ADR-0006：XMBC PRESS/RELEASE 为一次性按键状态前缀

## 状态

已接受，替代此前将 `PRESS/RELEASE` 视为持续模式的解析规则。

## 背景

早期实现假定 `{PRESS}` 或 `{RELEASE}` 会持续作用于后续一串键，直到另一个状态标记出现。用户提供的实际 XMBC 序列使用 `{PRESS}e` 和 `{RELEASE}e` 配对，随后直接书写 `3`、`e`、`q` 及辅助键组合。持续模式假设导致这些后续普通键全部成为仅松开事件，并产生大量 `KEY_UP_WITHOUT_DOWN`。

## 决策

- `{PRESS}` 与 `{RELEASE}` 只作用于紧随其后的一个普通键、扩展键或扫描码。
- 目标键被消费后立即清除显式状态，后续无前缀键恢复为普通 Down/Up 点击。
- `HOLD/HOLDMS` 与 `{PRESS}` 组合时在按下后插入延时；与 `{RELEASE}` 组合时在释放前插入延时。
- 鼠标继续使用自身 `D/U` 后缀表达显式状态，不继承键盘前缀。

## 后果

实际 XMBC 宏中的显式 E 按下/释放与后续普通点击能同时保持平衡。旧测试中假设单个前缀跨多个键的行为不再支持；需要逐键重复写入状态前缀。
