# ADR-0007：宏库使用真实文件夹与本地元数据

## 状态

已采纳，2026-07-12。

## 背景

用户现有雷蛇 XML 按人物文件夹保存，XMouse 文本则集中在记事本中。宏库需要改善查找、复用和备份，同时保持绿色、离线、可由资源管理器直接访问，不得把用户资料锁入专有数据库。

## 决策

- 宏库根目录由用户在设置中选择；根目录文件属于“未分组”，一级子文件夹直接映射同名分组。
- `.txt` 与 `.xml` 独立管理并在界面分栏，不按同名自动配对；其他附件不进入列表且不被修改。
- 收藏、最近使用和回收站放在根目录隐藏的 `.xmacrobridge/`，元数据只保存相对路径和时间。
- 外部文件默认复制入库；转换区使用工作副本，只有显式“保存到宏库”才新建或更新文件。
- 路径切换不移动内容；迁移只复制到空目录，逐文件校验 SHA-256 后才切换，源目录始终保留。
- 应用设置仅在 `%LocalAppData%/XMacroBridge/settings.json` 保存当前根路径；自动化必须改用临时路径。

## 结果

宏库可脱离应用直接备份和整理，并能接入现有目录而无需重排。真实文件夹重命名会同时影响其中附件，因此 UI 必须提前提示。相对路径元数据需在重命名和移动时同步更新；损坏元数据不得阻止普通文件继续显示。
